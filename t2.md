# 基于三路解耦混合注意力的视频生成加速方案
Title: Tri-Branch Hybrid Sparse Attention for Efficient and Consistent Video Generation1. 

## 核心摘要 (Executive Summary)

针对视频生成模型（如 DiT, Video LDM）在处理长视频（>5s, 720P+）时面临的显存爆炸与时空不一致难题，我们提出了一种 Tri-Branch Hybrid Attention（三路混合注意力） 架构。该架构将注意力计算解耦为三个正交的维度：Local（局部物理连续性）、Sparse（长程语义一致性） 和 Global（全局环境一致性）。通过引入 Global Compressed Attention (KV-Pooling) 替代全量 Attention，并结合 Fused Index Injection 技术，我们在保持 $O(N)$ 线性复杂度的同时，实现了优于滑动窗口（Sliding Window）的生成质量，彻底解决了长视频中的背景闪烁与物体 ID 漂移问题。

## 架构设计细节 (Architecture Design)
建议采用 加权并行三分支 (Weighted Parallel Tri-Branch) 结构：
$$\text{Attn}(Q, K, V) = \text{Local}(Q, K, V) + \text{VSA}(Q, K, V) + \text{Global}(Q, K_{pool}, V_{pool})$$

| 分支名称          | 核心作用                                | 推荐实现配置                                                          | 算力占比 (估) |
|---------------|-------------------------------------|-----------------------------------------------------------------|----------|
| Local Branch  | 防抖动：保证 t 与 t±1 的物理平滑，消除 Block 边缘拼缝。 | Cross/Star Window (十字形滑窗)。不要用全 3×3×3 立方体，只取时间轴和空间轴的相邻 Block。    | ~15%     |
| Sparse Branch | 保一致：捕捉长程 ID（如 t=0 的人脸），解决“变脸”问题。    | VSA Top-K。基于 Block Mean Pooling 选 Top-8 或 Top-16。               | ~75%     |
| Global Branch | 稳背景：统一光照、色调、氛围，消除背景闪烁。              | Compressed KV。保持 Q 全尺寸，对 K/V 做 64× (Block Size) 的 Mean Pooling。 | ~10%     |


## 工程落地要点 (Implementation Checklist)

这是 Infra 工程师必须死磕的 "Do's and Don'ts"，也是你绩效的来源： 

### ⚠️避坑指南 (Critical Pitfalls)
* 绝对禁止 Repeat 操作：Global 分支算完后，严禁将结果复制（Repeat）回原尺寸相加。这会导致严重的 马赛克效应 (Block Artifacts)。
  * 正解：直接计算 $Attention(Q_{full}, K_{pool})$，输出即为全分辨率。
* 拒绝显式 Permute/Reshape：不要在显存里把视频重排成 [N_blocks, 4, 4, 4, C]。这会消耗 50% 以上的显存带宽且无计算产出。
  * 正解：写 Fused Kernel，在加载数据到 SRAM 时，利用数学公式计算坐标偏移，直接读取原始布局的数据。
* 避免 Atomic Random Gather：尽量不要让每个 Head 选完全不同的 Block，这会导致 DRAM Row Conflict。
  * 正解：在可能的情况下，强制 Head-Shared Selection（所有 Head 读相同的 Block），以实现 Memory Coalescing（合并访存）。
  
### 🚀 优化策略 (Optimization Tactics)
* Index Injection (索引注入)：不要写两个 Kernel（一个 Local 一个 Sparse）。在进入 Kernel 前，在 Python 层或轻量级 Kernel 中做 Final_Indices = Unique(TopK_Indices + Local_Indices)。
  * 收益：零 Kernel 启动开销，统一流水线。
* 算力与显存的权衡：在 720P/5s 场景下，VSA 稀疏度控制在 10% 左右，Local Window 控制在 2k token 以内。Global 分支可以直接复用 VSA Router 计算时生成的 Centroids (Pooled KV)，进一步省去一次 Pooling 计算。

## 预期收益 (Expected ROI)
* 显存效率：支持推理时长从 2s 提升至 10s+ (720P) 而不 OOM。
* 计算加速：相比 Full Attention，理论 FLOPs 降低 80% - 90%。
* 生成质量：相比单纯 Window Attention，长程

一致性（ID Retention） 显著提升；相比单纯 VSA，画面平滑度 显著提升。这份摘要结合了学术理论与你最关心的工程落地细节，既显得“高大上”（有理论深度），又显得“接地气”（能解决实际 Infra 痛点）。
